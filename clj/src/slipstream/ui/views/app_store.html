App Store Template

Layout:

  • 4 apps per row on big computer screens (.col-lg-3)
  • 3 apps per row on small computer screens (.col-md-4)
  • 2 apps per row on tablets (.col-sm-6)
  • 1 app per row on phones (nothing on xs)

As image examples you might use dummyimages like:

  • http://dummyimage.com/400x200/aa0000/fff.png
  • http://dummyimage.com/100x200/aa0000/fff.png
  • http://dummyimage.com/200x150/aa0000/fff.png
  • http://dummyimage.com/75x75/aa0000/fff.png

Usage: 

      [
        {
          :img-url ""
          ;; Will go to: $(".ss-app-thumbnail-container")
                            .css("background-image:
                                  url(app_image.png),
                                  url(images/app_image_placeholder.png);")

          :name ""
          ;; Will be the content in .ss-app-name

          :uri ""
          ;; Will go to:  $("a[href='#'].ss-app-name-container")
                            .attr('href', '/modules/app/version')

          :version ""
          ;; Will be the content in ss-app-version-number.

          :price ""
          ;; Will be the content in .ss-app-price or nil for free.

          :publisher ""
          ;; Will be the content in .ss-app-publisher

          :description ""
          ;; Will be the content in .ss-app-description
        }

        ...

      ]

Label decorators for New and Updated: 

      [
        {
          :decorator :new
          ;; Will $(".ss-label-decorator-new").removeClass("hidden");

          ...
        }

        {
          :decorator :updated
          ;; Will $(".ss-label-decorator-updated").removeClass("hidden");

          ...
        }

        {
          :decorator nil
          ;; Will leave the app thumbnail without decorator.

          ...
        }

        ...

      ]

Note: in case both are enabled, New wins. (If something is new, it can be somehow considered updated as well.)
