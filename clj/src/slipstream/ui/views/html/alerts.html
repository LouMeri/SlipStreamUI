Generic alerts

Content can be modified. To be passed to the (base) function as follows: 

     :alerts [{:type :error
     ;; You might add the following flag to any alert type to disable the closing button:
     ;; :dismissible false
     :msg "Something is wrong with your credentials."}

     {:type :warning
     :msg "With the deployment you just performed you are aproaching the quota limit on Amazon EC2 cloud."}

     {:type :success
     :msg "Your credentials for Amazon EC2 cloud were successfully validated."}

     {:type :info
     :msg "To have an overview of your deployments go to the dashboard."}]

Error

Something is wrong with your credentials.

Warning

With the deployment you just performed you are aproaching the quota limit on Amazon EC2 cloud.

Success

Your credentials for Amazon EC2 cloud were successfully validated.

Info

To have an overview of your deployments go to the dashboard.

Prebuilt alerts

Content cannot be modified. To be passed to the (base) function using the :#id selector. They can be mixed with the generic alerts: 

   :alerts [{:type :error, :msg "Blah blah blah..."}
   :#alert-wrong-credentials]

Error

Something is wrong with your credentials. Do you need a new password?

Dynamic alerts

Alerts can be dynamically added to a page with the following function call. You can try it in the JavaScript console of this template page. 

    ss_alert({type: "error", title: "Dynamic alert", msg: "This is the message."});
